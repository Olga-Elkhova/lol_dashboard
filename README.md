# 🎮 League of Legends аналитический дашборд

Аналитический дашборд для визуализации статистики рейтинговых игр League of Legends. Проект собирает данные через официальное API Riot Games, обрабатывает их и представляет в виде интерактивных графиков.

---

## О проекте

Дашборд отображает статистику матчей, игроков и чемпионов League of Legends за текущий месяц для регионов Европа (EUW) и Северная Америка (NA).

**Собранные данные:**
- 131 532 матчей
- 88 459 игроков
- 172 чемпиона

**Стек технологий:**
- **Сбор данных:** Python, Requests, Pandas
- **Хранение:** Parquet (колоночный формат, сжатие в 10-15 раз по сравнению с CSV)
- **Аналитическая база:** DuckDB (быстрые SQL-запросы к Parquet)
- **Визуализация:** Streamlit + Plotly
- **Управление зависимостями:** pip + requirements.txt

## 🗄️ Описание таблиц данных

### 1. `players_data.parquet` — Информация об игроках

Содержит данные о всех игроках из лиг Challenger, Grandmaster и Master за текущий месяц.

| Поле | Тип | Описание |
|------|-----|----------|
| `puuid` | string | Уникальный идентификатор игрока (первичный ключ) |
| `summoner_name` | string | Имя игрока в игре |
| `region` | string | Регион: `euw1` (Европа) или `na1` (США) |
| `tier` | string | Ранг игрока: `challenger`, `grandmaster`, `master` |
| `league_points` | int | Текущее количество очков лиги (LP) |
| `total_wins_season` | int | Всего побед в сезоне |
| `total_losses_season` | int | Всего поражений в сезоне |
| `matches_played_month` | int | Количество матчей за текущий месяц |
| `wins_month` | int | Побед за месяц |
| `winrate_month` | float | Винрейт за месяц (%) |
| `avg_kills` | float | Среднее количество убийств за матч |
| `avg_deaths` | float | Среднее количество смертей за матч |
| `avg_assists` | float | Среднее количество вспоможений за матч |

**Пример данных:**

region: euw1

tier: challenger

league_points: 1947

matches_played_month: 216

winrate_month: 50.46

avg_kills: 4.06

avg_deaths: 3.52

avg_assists: 7.67

### 2. `matches_data.parquet` — Информация о матчах

Содержит данные о каждом сыгранном матче.

| Поле | Тип | Описание |
|------|-----|----------|
| `match_id` | string | Уникальный идентификатор матча (первичный ключ) |

| `game_duration_sec` | int | Длительность матча в секундах |
| `game_creation` | int | Временная метка начала матча (Unix, мс) |
| `game_version` | string | Версия игры (патч) |
| `blue_team_won` | bool | Победила ли синяя команда (`True` / `False`) |

**Пример данных:**

region: euw1

game_duration_sec: 1732

game_creation: 1781052926104

game_version: 16.11.782.9736

blue_team_won: False

Содержит агрегированную статистику по каждому чемпиону.

| Поле | Тип | Описание |
|------|-----|----------|
| `champion_name` | string | Имя чемпиона (первичный ключ) |
| `champion_id` | int | Числовой ID чемпиона |
| `matches_played` | int | Количество матчей с этим чемпионом |
| `wins` | int | Количество побед |
| `winrate` | float | Винрейт (%) |
| `avg_kills` | float | Среднее убийств за матч |
| `avg_deaths` | float | Среднее смертей за матч |
| `avg_assists` | float | Среднее вспоможений за матч |
| `total_kills` | int | Общее количество убийств |
| `total_deaths` | int | Общее количество смертей |
| `total_assists` | int | Общее количество вспоможений |

**Пример данных:**

## Установка и запуск

### 1. Клонирование репозитория

git clone https://github.com/Olga-Elkhova/lol_dashboard.git
cd lol_dashboard

### 2. Установка зависимостей

pip install -r requirements.txt

### 3. Настройка API ключа (опционально)

Если вы хотите собрать свежие данные:

- Зарегистрируйтесь на developer.riotgames.com

- Получите Development API Key

- Создайте файл .env в корне проекта:

RIOT_API_KEY=RGAPI-ваш_ключ

### 4. Сбор данных (опционально)

Если вы хотите собрать свежие данные:
python data_pipeline.py

### 5. Конвертация в Parquet (опционально)

Для ускорения загрузки в дашборде:
python convert_to_parquet.py

### 6. Запуск дашборда

streamlit run app.py

## Функциональность дашборда

Дашборд содержит 5 вкладок с разными аналитическими срезами.

### 1. Общая статистика

Базовый обзор данных с ключевыми метриками.

- Распределение LP	- Гистограмма очков лиги (LP) среди игроков

- Длительность матчей -	Boxplot распределения длительности игр

- Ключевые метрики -	Средний LP, средний винрейт, средняя длительность, общее количество матчей

**Цель:** Быстрая оценка общего состояния данных.

### 2. Анализ чемпионов

Анализ меты чемпионов.

- Топ-10 по винрейту -	Лучшие чемпионы по проценту побед (мин. N матчей)
- Флоп-10 по винрейту -	Худшие чемпионы по проценту побед
- Нишевые чемпионы -	Редкие чемпионы (100-500 матчей) с высоким винрейтом

**Цель:** 

- Выявление сильных и слабых чемпионов для балансировки патчей

- Поиск скрытых "секретных" пиков для контента и стратегий

### 3. 🌍 Сравнение регионов

Сравнение метрик между EUW (Европа) и NA (Северная Америка).

- Винрейт синей команды -	Сравнение процента побед синей команды по регионам
- Средняя длительность - Сравнение средней длительности матчей по регионам

**Цель:**

- Понимание региональных особенностей меты

- Маркетинговые инсайты для разных аудиторий

### 4. Аномалии LP и винрейта

Выявление выбросов и подозрительных аккаунтов.

- Аномалии LP -	Игроки с аномально высоким или низким LP (через IQR)
- Аномалии винрейта -	Игроки с винрейтом >80% или <20%
- Таблица аномалий -	Список подозрительных аккаунтов с их метриками

**Цель:**

- Выявление бустеров или читеров (аномально высокий винрейт)

- Обнаружение багов в системе начисления LP

- Модерация и контроль честности игры

### 5. Кластеризация игроков

Группировка игроков по стилю игры на основе KDA (убийства/смерти/помощи).

- 3D-визуализация кластеров -	Scatter-plot с проекцией трёх метрик (убийства, смерти, помощи)
- Средние показатели кластеров -	Таблица со средними значениями KDA для каждого кластера
- Распределение по кластерам -	Pie chart с долей игроков в каждом стиле
- Примеры игроков -	Список игроков из каждого кластера

**Кластеры:**

- Кластер 0: Агрессивные игроки (много убийств и смертей)

- Кластер 1: Поддерживающие игроки (много помощи, мало смертей)

- Кластер 2: Сбалансированные игроки (средние показатели)

- Кластер 3: Осторожные игроки (мало смертей, мало убийств)

**Цель:**

- Сегментация аудитории для маркетинговых кампаний

- Понимание популярных стилей игры

- Балансировка чемпионов под разные стили

### Фильтры

- Регион -	Переключение между EUW, NA и всеми регионами
- Минимум матчей -	Настройка минимального количества матчей для включения чемпиона в анализ

## Технические детали

- Сбор данных	- Python + Requests + Pandas
- Хранение	- Parquet (колоночный формат, сжатие 10-15x)
- Аналитическая БД	- DuckDB
- Визуализация	- Streamlit + Plotly
- Кластеризация	- Scikit-learn (K-Means)

## Rate Limiting

Сборщик данных учитывает ограничения Riot API:

- 20 запросов в секунду

- 100 запросов за 120 секунд

Реализован планировщик с очередью времени для автоматического ожидания.

## Формат хранения

Parquet обеспечивает:

- Сжатие данных в 10-15 раз по сравнению с CSV

- Быстрые колоночные чтения (читаются только нужные колонки)

- Нативную поддержку в DuckDB

## DuckDB

DuckDB используется для:

- Быстрых SQL-запросов к Parquet-файлам

- Агрегации данных без загрузки всего датасета в память

- Интеграции с Pandas для визуализации

## Требования

- Python 3.9+

- Оперативная память: от 4 ГБ

- Место на диске: от 500 МБ
